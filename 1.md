# 连锁零售门店库存管理系统后端管理文档
## 一、系统架构
### 1.1 技术栈选型
| 层级         | 技术选型                          | 说明                                                                 |
|--------------|-----------------------------------|----------------------------------------------------------------------|
| 后端框架     | Spring Boot 2.7.x + Spring MVC   | 快速构建企业级应用，提供RESTful API支持                             |
| 数据库       | MySQL 8.0                         | 存储业务数据（门店、商品、库存、订单等）                             |
| 缓存         | Redis 6.x                         | 缓存用户登录状态、热点库存数据，提升系统性能                         |
| 消息队列     | RabbitMQ 3.12.x                   | 异步处理库存更新、订单同步等耗时操作，解耦业务模块                   |
| ORM框架      | MyBatis-Plus 3.5.x                | 简化数据库操作，支持代码生成、分页查询、逻辑删除等功能               |
| 接口文档     | Knife4j 3.x（基于Swagger）        | 自动生成在线接口文档，支持接口调试                                   |
| 安全框架     | Spring Security + JWT              | 实现用户认证、角色权限控制                                           |
| 监控         | Spring Boot Actuator + Prometheus | 监控系统运行状态、接口性能、数据库连接等                             |

## 二、程序流程图
### 2.1 采购入库流程
```mermaid
flowchart TD
    A[创建采购入库单] --> B{选择供应商/门店}
    B -->|必填项校验| C[添加商品明细]
    C --> D{校验商品信息}
    D -->|效期/批次合规| E[提交入库单]
    E --> F{角色权限校验}
    F -->|店长/总部管理员| G[审核入库单]
    G -->|审核通过| H[扣减Redis缓存]
    H --> I[更新MySQL库存表]
    I --> J[记录操作日志]
    J --> K[入库完成]
    G -->|审核不通过| L[退回修改]
    L --> A
    D -->|效期异常| M[提示错误]
```

### 2.2 销售出库流程
```mermaid
flowchart TD
    A[接收线上/线下订单] --> B{校验订单信息}
    B -->|信息完整| C[锁定库存（Redis）]
    C --> D{库存充足?}
    D -->|是| E[创建销售出库单]
    E --> F[扣减库存（MySQL）]
    F --> G[释放锁定库存]
    G --> H[记录操作日志]
    H --> I[出库完成]
    D -->|否| J[返回库存不足提示]
    B -->|信息缺失| K[提示参数错误]
```

### 2.3 跨店调拨流程
```mermaid
flowchart TD
    A[创建调拨单] --> B{选择调出/调入门店}
    B -->|门店不同| C[添加商品明细]
    C --> D{校验调出门店库存}
    D -->|库存充足| E[提交调拨单]
    E --> F{审核权限校验}
    F -->|总部管理员| G[审核调拨单]
    G -->|通过| H[异步更新双方库存]
    H --> I[调出门店扣减]
    H --> J[调入门店增加]
    I --> K[记录操作日志]
    J --> K
    K --> L[调拨完成]
    G -->|不通过| M[退回修改]
```

#### （2）请求封装 `common/request_util.py`
```python
import requests
from config.settings import BASE_URL

class RequestUtil:
    def __init__(self):
        self.base_url = BASE_URL
        self.session = requests.Session()
        self.token = None

    def set_token(self, token):
        self.token = token
        self.session.headers.update({"Authorization": f"Bearer {token}"})

    def send_request(self, method, url, **kwargs):
        full_url = f"{self.base_url}{url}"
        try:
            response = self.session.request(method=method, url=full_url, **kwargs)
            response.raise_for_status()  # 抛出HTTP错误
            return response.json()
        except requests.exceptions.RequestException as e:
            print(f"请求失败: {e}")
            return None

# 全局实例
request_util = RequestUtil()
```

#### （3）Token获取 `common/auth_util.py`
```python
from common.request_util import request_util
from config.settings import ADMIN_USER, STORE_USER

def get_admin_token():
    """获取总部管理员Token"""
    response = request_util.send_request(
        method="POST",
        url="/auth/login",
        json=ADMIN_USER
    )
    if response and response["code"] == 200:
        token = response["data"]["token"]
        request_util.set_token(token)
        return token
    raise Exception("获取管理员Token失败")

def get_store_token():
    """获取门店店长Token"""
    response = request_util.send_request(
        method="POST",
        url="/auth/login",
        json=STORE_USER
    )
    if response and response["code"] == 200:
        token = response["data"]["token"]
        request_util.set_token(token)
        return token
    raise Exception("获取门店Token失败")
```

from common.auth_util import get_store_token, get_admin_token
from config.settings import STORE_USER

# 前置条件：获取Token
@pytest.fixture(scope="module", autouse=True)

@pytest.mark.parametrize("test_data", test_inbound_data, ids=lambda x: x["name"])

    response = request_util.send_request(
        method="POST",
        url="/inventory/inbound",
        json={
            "supplierId": test_data["supplier_id"],
